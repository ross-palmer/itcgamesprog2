Add the following functionality to introduce some gameplay to the tank demo: (marks for each section is shown inside square brackets).

  1. Implement projectile firing. It should be possible to fire a projectile at a rate of 1 per second. The projectile should not pass through a wall. A speed value of about 1000 works well, don't forget to use the delta time (dt) when translating the projectile (hint: see how we use delta time for moving the Tank in Tank::update()). Tip: For the firing rate, use a clock timer. If you use a simple counter, the timing will be machine speed dependent (not accurate). [15]

  2. Add a countdown timer of 60 seconds after which the game should stop. Show the value of the timer on a HUD (if using a floating point variable, don't show decimal places on the HUD). [10]

  3. Draw the player's tank in one corner of the screen, where the corner is picked randomly. Add pop-up targets - you decide on the duration they stay on screen. The target properties should be defined in the yaml file i.e. duration, position, rotation etc. The targets should not always appear in the same position and should not appear overlapping a wall. For example, in the yaml file you could define a targets position as:

    position: {x: 50, y: 300, randomOffset: 20}

    where a maximum random offset of up to +20 or -20 is applied to it's x and y position. Targets should appear in sequence (one by one) and not all together. [15]

  4. The turret is broken, it drifts (rotates) slightly whenever the tank is moving. The idea is to make it more challenging to line up a shot. [5]

  5. A warning should be given to the player to indicate a target is about to disappear. For example, the target may flash on and off. [10]

  6. Targets should disappear when hit by a projectile from the tank and the player's score should be updated on the HUD. If the player strikes a target, the next target should appear immediately and the previous target's remaining time should be added to the new target as a reward. It should be possible (but challenging) to clear all targets in a game. [15]

  7. At game over, report the player's score (number of targets hit) and accuracy. For example, if 20 shots were fired, but only 10 targets hit, the accuracy is 50%. Tip: Check your accuracy reporting is correct (count your shots fired and registered hits). The game should also keep track of the player's best peformances, i.e. the current best score/accuracy should be saved in the YAML file. Show the current score/accuracy and previous best score/accuracy at game over. [10]

  8. Extras: [20]

    Add something interesting to the project such as one item from the list below. Note, if you have time to spare, consider doing more than one, as this should be one of your portfolio pieces from second year.

    • Add mouse support for controlling the tank. Right click will move the tank to the indicated location. Left click will fire. Pressing the mouse wheel causes the turret to track the movement of the mouse pointer.
    • Add particle effects for firing, target hit. Reource Link.
    • Use the Thor library to add animations to the project. Resource Link.
    • Add a larger map that scrolls as the player tank moves around. If a target appears off-screen, draw an arrow on the HUD to show the direction the player should move. Resource link for sf::View. Resource link for drawing arrows
    • Create a two player mode with controller support, where two players can duel it out on the one screen.
    • Create a simple level editor where the player can place wall sprites, targets etc on the map. The editor should write to a YAML file (I will provide some assistance with this aspect) which is then used by the game.
