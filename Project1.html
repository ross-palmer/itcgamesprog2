The brief is to create an initial version of a cross between Space Invaders and Arkanoid that you will improve later in Mini Project 2. In this project, the goal is to clear the level of enemy invaders within a defined time period. The enemy invaders will move horizontally only (left/right) and will be protected by a row of blocks underneath. This pattern will be repeated to give four rows in total (invaders, blocks, invaders, blocks). The player will control a paddle that will deflect a laser bolt up towards the bricks/invaders. A collision between the bolt and brick or invader will destroy the target and result in a deflection of the bolt. Note: for this project, you can use a circular shape for the bolt, as a 'baton' shaped bolt would need to be rotated upon deflection.

Functionality and marking scheme

  1. Create four rows of invaders/bricks. There should be one row of 6 invaders with another row of bricks underneath that to cover the width of the screen. Repeat this pattern to give four rows in total. You can decide on the initial placement of the invaders, but they should all move in unison either towards the left or right side of the screen and then reverse direction. If for example the invaders are moving left, they keep moving in that direction until the leftmost invader reaches the leftmost screen position and then they all reverse direction. This process is repeated. [15]

  2. At the start of the game, a laser bolt should drop at an angle towards the player paddle. The bolt should be deflected off the paddle as follows. Assume the length of the paddle is l and the distance of the bounce position of the bolt on the paddle from one edge is d. The bolt should be reflected by 90 degrees when it hits the exact middle and 45 degrees from the very edges. The points between these should interpolate linearly between these values. To calculate the deflection angle, we do it in 2 steps.

    Step 1: Divide d with l, subtract 0.5 from it:

    deflectionAmount = (d / l - 0.5)

    Step 2: Multiply 90 by the absolute value of deflectionAmount and subtract the result from 90 degrees:

    angle = 90 - (90 * abs(deflectionAmount))

    If for example, the paddle length is 10, and the bolt strikes the exact middle of the paddle, then we get:

    deflectionAmount = ( 5 / 10 - 0.5 ) = 0
angle = 90 - ( 90 * abs(0) ) = 90 - 0 = 90 degrees

    To take a second example, if the bolt strikes the very left edge of the paddle, then we get:

    deflectionAmount = ( 0 / 10 - 0.5 ) = -0.5
angle = 90 - ( 90 * abs(-0.5) ) = 90 - 45 = 45 degrees

    The angle should be converted to a direction vector and applied to the bolt:

    x = cos(angle) * speed
y = sin(angle) * speed

    Note that you will need to retain the sign (+ or -) of the x component of the bolt's direction vector so you know which way to bounce the bolt. [10]

  3. Paddle movement and bolt speed. Use the left/right arrow keys for paddle movement (paddle should not move off-screen, but stop when it reaches the extents of each side). Use the up arrow key to grant the player a bolt "power-up" - this is temporary increase in bolt speed. Use playtesting to determine the appropriate time value. If the bolt goes off-screen, it should reset at the normal speed. Allow the player 3 uses of this feature. [10]

  4. Bolt to wall collisions. Use the principle that the angle of incidence is angle of reflection. Assume three walls in the game - left side, right side and top. If the bolt collides with the top wall, reverse the y component of the bolt's direction vector. [10]

  5. Bolt to brick collisions. Upon collision, the brick will disappear. If the bolt collides with the bottom of a brick, the bolt should deflect downwards. If the bolt collides with the top of a brick, deflect upwards. If the bolt collides with either the left or right edge of a brick, allow the bolt continue in it's y direction of travel. In all cases, again use the principle that the angle of incidence is angle of reflection. Bolt to invader collisions should work on the same principle as bolt to brick collisions. [10]

  6. Timer, HUD and game states. Add a target time by which all invaders should be destroyed - you will need to playtest to determine a suitable time. Show the value of the timer on a HUD (if using a floating point variable, don't show decimal places on the HUD). If the bolt passes below the position of the paddle it is out of bounds. In this case, apply a small time penalty and reset the bolt to it's starting position. The HUD should also show the number of bolt power-ups remaining. Add a simple game win/lose screen and the possibility to play again. [20]

  7. All on-screen entities (paddle, bricks, invaders) should have their position defined in a yaml file, along with the game level duration (this will be a value in seconds). Position the paddle on the lower part of the display. [10]
