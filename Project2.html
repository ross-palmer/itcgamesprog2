Following on from the work you have completed in lab 8, extend the tank demo to add the following functionality.

  1. Add a basic state machine to the AI tank to enable other behaviours. Add an AIState enum to the TankAI class as follows:

    enum class AIState { PATROL_MAP, SEEK_PLAYER, SPAWN_ALLY };

    These will represent the three states for the AI tank which will determine it's behaviour. The AI should start in PATROL_MAP mode, patrolling the map and avoiding obstacles. If the AI tank detects the player tank (see point 2 below), it will switch state to SEEK_PLAYER. In this mode, the AI tank will gain a slight increase in speed so it moves faster than the player. If an AI tank touches the player tank at any stage, then the game is over. [25]

  2. The Player tank leaves a heat haze behind as it traverses the map which slowly dissipates. The AI tank has IR vision and can only detect the player by this heat signature. To simulate this idea, every 100ms the player tank will add an invisible sf::CircleShape at it's current position to a std::list, but only if it is moving. The list should hold a maximum of 30 positions (circles) only, where the newest position is always at the front, and the oldest at the rear of the list. Every update, the AI tank will check if it's ahead vector intercepts any of the Circle shapes in this list. If so, it will then "know" the player position and seek out the player tank. [25]

  3. The player tank has a finite number of projectiles. If a projectile hits an AI tank, the AI shrinks to half it's size, and it's movement speed is halved accordingly. The player must now touch the shrunken AI tank to eliminate it. This effect is only temporary after which the AI tank regains it's normal size and resumes in PATROL_MAP mode. The player must eliminate three AI tanks to complete the level. You decide on appropriate values here for projectiles and shrink duration. [20]

  4. The AI tank can spawn a total of four allies, one at a time. You decide on when this should happen and the total upper number of allies that can be spawned (this should bear some relationship to the number of projectiles that player has). When spawning an ally, the AI switches to SPAWN_ALLY mode, and stops moving. Ally tanks have the exact same behaviour as the the main AI tank, but you cannot exceed the maximum number of spawned tanks. [10]

  5. Add a debug mode that is activated by a key press. In this mode, draw the ahead and ahead2 vectors. Also draw the circle shapes that represent obstacles, and use a different colour to show the circle shapes that depict the trail of the player tank. [10]

  6. A game time of 1 minute applies. Use a HUD to show any relevant information (e.g. game time, remaining projectiles, number of AI eliminated, remaining number of spawnable AI tanks). [10]

  7. Bonus: If you complete this section, you can "top up" your project mark up to a maximum of 100. See if you can use a shader to add a heat haze effect behind the player tank (the area covered by the shader will be determined by the last position in the list maintained by the player tank). Example: https://github.com/SFML/SFML/wiki/Source%3A-HeatHazeShader [20]
