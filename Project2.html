Following on from the work you have completed in lab 8, extend the ECS tank demo to add the following functionality:

  1. Define an imaginary track that the AI tank must follow around the map.

    1.1 Start by reading this article on Path Following.

    1.2 Define 20 nodes in the yaml file which should be sufficient to form some kind of racetrack pattern around the map. Leave the obstacles in place as the tank should steer around these while trying to move to the next node. You will need to parse the data from these nodes inside the LevelLoader. [20]

    1.3 You will need to define a NodeCreator class in EntityCreator.cpp to create each entity to represent a Node. Hint: Look at how the WallCreator class works. NodeCreator is then used inside the LevelSystem (again, see how WallCreator is used in LevelSystem.cpp). [10]

    1.4 Inside the TankAI class, modify the AI tank behaviour so it now follows the defined path. Consider adding another state to the TankAI such as AiBehaviour::PATH_FOLLOWING [40]

  2. Add a simple HUD to show the next waypoint number the tank is heading for and also the time taken for the tank to complete one circuit. Hint: The next waypoint number should be a property of TankAI. Reset the timer when it has finished one complete lap. The HUD should be implemented as an EntityX system (i.e. HUDSystem). Hint: Have a look at the definitions of other systems first before attempting to write this one. A HUD is not a game object, so there is no need to create a HUD Enitity nor HUD components. [30]
