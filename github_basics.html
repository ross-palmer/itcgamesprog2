A quick guide to using Github

In brief, Git is a source code management system, while Github is a cloud based hosting repository for Git projects. You are going to use Github to host your Programming practicals as you complete them throughout the year. If you do not have a github account, you create one here using your IT Carlow email address.

  1. Join the Programming year 2 assignment that I have setup on github. Open the URL https://classroom.github.com and login using your github credentials. Next, open the following URL in a new browser tab: https://classroom.github.com/a/rVpnCVJq. Follow the prompts and this will create an empty repository for you.
  2. Open the command line tool Git Bash - this will enable you execute common git commands.
  3. Type the command cd G:\ and press return. This means change directory (cd) to the root of your G: drive.

  4. You now need to tell Git who you are with these two commands

    git config --global user.name "Your name here"

    (press return)

    git config --global user.email "Your email address@itcarlow.ie"

    (press return)

  5. Clone your new empty repository. This means it will be downloaded to your local (G:) drive from where you can begin adding files to it. The clone command is:

    git clone https://github.com/ITCGamesProg2/lab19-youremail.git

    where youremail is first part of your email address (i.e. up to the @ sign) you used to sign up to github.

    Rename your repository so that it is formatted as follows:

    https://github.com/ITCGamesProg2/labs19-firstName-lastName.git

    For example: https://github.com/ITCGamesProg2/labs19-Jane-Bloggs.git

    Go to settings, then click in the box containing your repository name, replace your email address with (your) firstName-lastName, then click Rename.

  6. This will create a directory with the same name as your repository in your local filesystem, specifically the location (G:\) you specified in step 3.

  7. Next, using a web browser, open this URL:

    https://github.com/ross-palmer/SFML_Playground

    Click on the green 'Clone or Download' button and then choose 'Download ZIP'. Unzip the file and copy it's contents to your new repository folder on the G: drive. You can now open the Visual Studio solution and work on the project.

  8. When you have finished editing code, you need to commit your work back to your own repository. If you have modified any existing files (most likely) OR added any new files, you need to stage them for the first commit:

    git add .

    This adds all files in the current directory and any subdirectory to the staging area, thus preparing them to be included in the next git commit.

  9. Commit the files that you've staged in your local repository:

    git commit -m 'First commit'

    'First commit' is a commit string - this should generally summarise the changes you have made to the code base, so this will obviously change between commits.

  10. If you are working in lab A200, you will need to configure git to use the proxy server:

    git config --global http.proxy http://proxycw3.itcarlow.ie:8080

  11. Push the changes in your local repository to the remote (a.k.a origin) master branch on GitHub:

    git push origin master

Using Git correctly takes time and practice. There is an interactive site here where you can practice and learn more about git commands.

You can also find lots of online help with Github repositories here.
